Notes

use document.forms to collect all the form in the document. Even if there is only one form, it will still be a collection and require index notation.

while the submit() function can be used, button and input elements with the type ‘submit’ and inputs with the type ‘image’ can also be used to manually submit

buttons with type ‘reset’ will do the same as the function, though they are considered poor usability

certain events are specific to forms, such as the focus event, which is activated whenever the cursor is placed inside the element, and the blur event, which is activated when the cursor leaves the element

the change event happens when the cursor leaves an element after it has been changed.

a submit event can be used to intercept a form before the normal submission takes place, allowing it to be interacted with

different types of input elements: text(default), password, checkbox, radio button, hidden, file, and additional elements that have been added and are becoming supported

while putting type=’text’ for a text input is not necessary, it helps make the purpose of the input clear and is therefore recommended

giving more than one element the same name property allows them to be accessed as a collection

if a browser does not support a new input element then it will be displayed as a normal input

it is recommended to validate form both client side and server side, as the JavaScript code can be altered and the client side can even be skipped entirely

though HTML5 has its own validation API, not all browsers support it

JavaScript allows OOP through objects, referred to as prototype-based, but also includes classes

entering only one parameter into the Array constructor set the length property of the array to that parameter and is full of undefined

the constructor property can be used to create a copy of an object even when it is unknown what the name of the constructor was

the static keyword before a function name makes it so that only the class can call that function, not the objects created from the class

the prototype property allows creation of an object that all shares all properties with all instances of the class or constructor function it came from, effectively allowing manipulation of a class after it has been created

arrays and objects should never be created inside a prototype

to make something within a class private, simply declare it like normal instead of using this.propertyName = propertyValue

non-enumerable properties will not appear as properties of objects when their properties are displayed. This is done with built in functions so that they do not block out the user created ones

each property has various attributes associated with it: value, writable, enumerable, and configurable. these attributes are unable to be changed by assignment, with the exception of value, and thus require special functions to control

if a method lacks an important return value, it is suggested to return ‘this’ so that chaining is possible

be careful of using ‘this’ inside nested functions, as it can cause it to lose its scope

you can get a reference to a function within an object by simply not including the parenthesis
