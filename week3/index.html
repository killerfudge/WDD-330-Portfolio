Notes

When reverencing a variable contained within an object, using the ‘this’ keyword is more reliable

‘this’ is evaluated during runtime and depends on the current context, so a function using ‘this’ can be used by multiple objects – “if obj.f() is called, then this is obj during the call of f”

arrow functions do not have their own ‘this’ and so use the ‘this’ from the outer context

variables within objects are defined as key: value pairs, with comma separations, instead of variable = value like I am used to

All objects are able to be modified while the program is running, even if declared using ‘const’

objects are assigned to variables with everything contained within ‘{}’ after the ‘='

if the property key is the same as the variable name that goes with it, then you can just include the name

while dot notation is more common to access the values within an object, you can also use a string form of the name within []. this allows you to evaluate an expression and use it as a property key, and nonstandard property and method names require this notation

the value of a property is allowed to be a JavaScript expression

Symbol objects can be used as keys by enclosing the Symbol’s name in []

objects can contain other objects

all objects are assigned by reference

object literals can be parameters for functions, allowing the parameters to be passed by name and in any order

the DOM represents a html document as a grouping of nodes laid out in a tree

the DOM treats whitespace between tags as a node

when interacting with CSS using the DOM, any CSS property names separated with a ‘-‘ are instead written in camelCase

the recommended way to handle events is to use event listeners. Inline event handlers are not recommended since they mix the JavaScript with the markup

functions are not followed by () when inside the argument for event listeners, as that would cause the function to be called when the listener is created, instead of when the event happens

the function used in an event listener needs to be named in order to remove it

clicking on one element means that you are also clicking on all elements that contain that element, such as the list that contains an item clicked, or the overall body and html tags. As event move through the different tags, they are said to propagate, with bubbling referring to going from child to parent, and capturing from parent to child. bubbling is the default
