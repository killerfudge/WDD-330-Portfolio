Notes

Form validation should be performed both client side and server side

The three main reasons to validate forms are: 1) to get the right data in the right format. 2) to protect the usersâ€™ data. 3) to protect the application from malicious users

common form validation attributes: required, minlength, maxlength, min, max, type, pattern

when a form element meets the specified attributes, it is considered valid. valid elements match the :valid CSS pseudo-class, allowing styling specific to them. invalid elements match :invalid

the Constraint Validation API consists of methods and properties usable with most typical HTML elements used in forms

the novalidate form attribute prevents the browser from automatically validating the form, allowing the programming to validate as they so choose
